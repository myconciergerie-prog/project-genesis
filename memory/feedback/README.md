# memory/feedback/

Project-scoped collaboration rules and feedback. **Most feedback rules live in Layer 0** (`~/.claude/CLAUDE.md`) — this directory holds only bits that are specific to Genesis work.

## What Layer 0 already covers

- Additive auth only — never log out any account
- No new terminal or Chrome windows
- R2.1 worktree discipline, no shortcut under close pressure
- R9 language policy (three tiers: dev EN / prod docs FR+EN / app runtime FR+EN)
- Async mid-flow questions welcome
- Best-practice-at-date default — always surface SOTA via research or cache
- Anti-Frankenstein — features must be pain-driven
- Cross-project research sharing — pointer files to avoid redundancy
- Per-project SSH identity pattern
- GitHub API auth via `GH_TOKEN` env override

Read `~/.claude/CLAUDE.md` Hard rules and Workflow patterns sections for the full content.

## What would go here

Only Genesis-specific collaboration notes that don't generalize. Examples that might qualify over time:

- Preferred PR description format for Genesis skill PRs (if it diverges from the general pattern)
- Specific response format preferences for Genesis design sessions
- Recurring misunderstandings that only happen in Genesis context

Nothing currently exists here. Entries added as patterns emerge.
