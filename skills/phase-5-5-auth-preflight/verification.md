# Phase 5.5.5 — Verification and memory writers

After `three-probe-test.md` reports `ALL=ok`, Phase 5.5 renders a single health card, writes two canonical reference-memory files, and hands control back to the Genesis protocol for Phase 5. Nothing else happens — no opportunistic cleanup, no extra probes, no "while we're here" additions.

  [x] ssh -T git@github.com-{{PROJECT_SLUG}}       (Hi {{GITHUB_OWNER}}!)
  [x] gh api user                                   ({{GITHUB_OWNER}})
  [x] gh api repos/{{GITHUB_OWNER}}/{{REPO}}        ({{GITHUB_OWNER}}/{{REPO}}, admin=true)

Automation path used
  [x] Playwright automation: {{used|fell_back|opted_out}}

```

The three possible statuses (same pattern as Phase -1.6):

- **READY FOR PHASE 5** — all three probes green, both reference files written.
- **READY FOR PHASE 5 WITH FALLBACKS** — probes green, Playwright opted out or fell back to paste-back for one or more steps. Phase 5 proceeds normally; the fallback record is informational.
- **BLOCKED — see {{PROBE_NAME}}** — a probe failed and cannot auto-recover. The card names the failing probe and re-invokes `three-probe-test.md` recovery for that probe only.

## Usage pattern

**SSH handles git** (clone, push, pull, fetch):

```bash
git clone git@github.com-{{PROJECT_SLUG}}:{{GITHUB_OWNER}}/{{REPO}}.git
git push origin <branch>
```

**`GH_TOKEN` env handles API** (`gh pr create`, `gh pr merge`, `gh api`):

```bash
GH_TOKEN="$GH_TOKEN" gh pr create --title "..." --body "..."
GH_TOKEN="$GH_TOKEN" gh api repos/{{GITHUB_OWNER}}/{{REPO}}
```

**Never `gh auth login`** — that would switch the global `gh` auth on this machine and affect every other project. Use env override, always.

## Idempotency

Re-running Phase 5.5 on a project where both reference files already exist is safe: the consent card detects the existing files and offers three options — *re-run from scratch* (with explicit rotation), *skip to Step 5.5.4* (re-verify only), or *cancel*. The default is *skip to Step 5.5.4*.

## Parsing the verification outcome

Claude reads the card's final status line and branches:

- **READY FOR PHASE 5**: emit a one-sentence confirmation, hand control to the Genesis protocol for Phase 5.
- **READY FOR PHASE 5 WITH FALLBACKS**: same, but the confirmation mentions which step fell back.
- **BLOCKED**: surface the blocker, invoke `three-probe-test.md` recovery for that probe, do NOT advance to Phase 5.

## Anti-Frankenstein reminders

- **Never treat a failed probe as "good enough"** — mark it either fixed or hard-blocked.
- **Never skip writing the reference files** — Phase 5 reads them; missing files break Phase 5's entry condition.
- **Never hold state in RAM beyond this card render** — everything Phase 5.5 knows must be in the two reference files.
- **Never add a "nice-to-have" probe to the three-probe list** — the list is fixed at three for a reason (the v1 spec's Learning 3).
