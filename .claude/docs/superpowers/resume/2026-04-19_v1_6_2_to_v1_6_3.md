## v1.6.3 scope — F5 primary + F1 + F4 runbook hardening

### F5 fix (mechanical, 8 files) — P0

Each of 8 SKILL.md files currently starts :

```
<!-- SPDX-License-Identifier: MIT -->
---
name: <skill-name>
description: ...
---
```

Claude Code's frontmatter parser requires `---` at line 1. **Fix options :**

**Option A** — move SPDX to trailing HTML comment (at end of SKILL.md, not start) :
```
---
name: ...
---
<rest of content>
<!-- SPDX-License-Identifier: MIT -->
```

**Option B** — remove SPDX from SKILL.md entirely, narrow R10.5 scope in `skills/genesis-protocol/rules/v1_rules.md` to source-code files only (.py / .ts / .md that are not plugin manifests). Cleanest option per R10.5 spirit (SPDX is for licensed source artefacts ; skill descriptions are metadata).

**Option C** — move SPDX to INSIDE the YAML frontmatter as a comment (YAML supports `# ...` comments). Verified acceptable by YAML spec, but verify Claude Code tolerates it.

Recommend **Option B** (scope-narrowing) — mechanical 8-file cleanup + one R10.5 doc edit. Deletes ~16 bytes/file × 8 = trivial net-negative change. Anti-Frankenstein compliant (removing unnecessary preamble).
