# v1.3.0 — Genesis Drop Zone Welcome Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v1.3.0 as a vertical slice of Genesis v2 LAYER A — a new sibling skill `genesis-drop-zone` that prints the welcome ASCII box, streams a reformulated acknowledgement of what the user gave, and closes with an honest bilingual bridge message. Extraction, `bootstrap_intent.md` write, and handoff to `genesis-protocol` are explicitly deferred to v1.3.1+.

**Architecture:** Sibling skill — a new directory `skills/genesis-drop-zone/` with two Markdown files (`SKILL.md` + `phase-0-welcome.md`) alongside the existing `genesis-protocol/`. Pure Markdown, no Python driver, no runtime plumbing, no writes outside the skill itself. Concentrated privilege is `none` (journal-system precedent). The new spec `v2_etape_0_drop_zone.md` (already committed in this worktree) is the 1:1 mirror target for `SKILL.md` — drift = merge-blocker per cross-skill-pattern #1.

**Tech Stack:** Claude Code plugin Markdown skills, SPDX headers, bilingual FR+EN runtime templates per R9 tier 3, `gh` CLI with R2.3.1 pre-flight, semver MINOR bump.

## Design decisions

### Why a sibling skill, not an extension of `genesis-protocol`

The vision doc `v2_vision_promptor_fusion.md` explicitly stratifies v2 into two layered protocols with distinct numbering: **LAYER A** (the conversation: Étape 0, 1, 2, 3) vs **LAYER B** (the engine: Phase -1, 0, …, 7). Folding Étape 0 into `genesis-protocol` (which is LAYER B pure today) would blur that split, break the existing 1:1 mirror between `genesis-protocol/SKILL.md` and `master.md`'s 7-phase table, and force the concentrated-privilege map to conflate two different privileges (engine-writes-project-directory vs conversation-captures-intent) under one skill. Sibling is the architectural-clean choice.

### Why v1.3.0 (MINOR), not v1.2.5 (PATCH)

New skill = backwards-compatible new functionality = MINOR per strict semver. v1.2.x was an F-fix cluster on v1.2.0 self-dogfood; v1.3.x opens a new line — "v2 conversational layer shipped by increments". Patching MINOR into PATCH accrues semver drift.

### Why a welcome-only vertical slice

Anti-Frankenstein gate: every line of plumbing landed in v1.3.0 without orchestration to justify it would be plumbing to re-migrate in v1.3.1. The surface-only slice produces a demonstrable user-visible artefact (invoke → welcome → acknowledgement → bridge) and nothing else. Extraction + `bootstrap_intent.md` write + `genesis-protocol` handoff are one coherent deferred scope unit shipped together in v1.3.1.

### R9 applied to a runtime-string-bearing skill

`phase-0-welcome.md` mixes tiers: structure + comments in English (dev-facing), runtime string templates coauthored bilingual FR+EN from day 1 (user-facing). This is the first Genesis skill to ship runtime strings — the tier pattern established here will be reused by every LAYER A skill that follows.

### Target self-rating calibration

Aspirational ≥9.3 on every axis; floor ≥9.0 on every axis. The two structural ceilings observed across v1.2.1→v1.2.4 — Prose stuck at 9.0 (verbose halt branches) and Best-at-date stuck at 8.6–8.8 (F-fix cycles don't generate new research) — are broken here by the vertical-slice discipline (compact prose) and the inline R8 citations of `v2_promptor_fusion_landscape_2026-04-17.md` (direct SOTA anchoring).
